# Visual Hierarchy Rules

Use this file when the render feels visually confusing even if content is correct.

## Reading Path

- Establish one primary path: title -> focal object -> interpretation -> source.
- Keep the protagonist visually dominant.
- Avoid competing centers of gravity.
- Use body bands and rails to make the order obvious.

## Emphasis

- Emphasize one thing at a time: value, row, bar, quadrant, path, quote, or decision.
- Use size, position, weight, and accent together sparingly.
- If everything is bold, nothing is emphasized.
- Weak evidence does not deserve strong emphasis.

## Contrast And Repetition

- Repetition is for comparison. Use the same grammar when the reader should compare similar
  objects.
- Difference should be visible through one controlled variable: color, weight, size, position,
  or label.
- Do not change both chart grammar and color meaning when only the data changes.

## Alignment And Whitespace

- Align objects to a spine: left edge, centerline, baseline, top edge, or chart axis.
- Use whitespace to separate roles, not to fill unused canvas.
- Prefer hairline rules and spacing over heavy boxes.
- If a slide has large unused zones, enlarge or reposition the proof before adding decoration.

## Typography

- The hierarchy is title > section heading > body > metric subline > source.
- Body proof must be readable. Footer can be small.
- Keep Japanese and English mixed text within stable line lengths.
- Do not scale font size with viewport or page novelty; use the token scale and layout repair.
- Metric value, unit, and YoY/delta are separate layers. Preserve line gap.

## Accessibility

- The main claim should be recognizable in thumbnail.
- Do not rely on color alone; pair color with label, position, or weight.
- Keep contrast strong enough for projectors.
