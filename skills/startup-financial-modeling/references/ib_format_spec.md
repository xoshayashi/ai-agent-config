# IBフォーマット仕様（生成器の実装契約）

出典: BIWS Excel Formatting Best Practices / Training the Street / Macabacus /
Financial Edge / FAST Standard / WSO ほか4ソース以上でクロスチェック済み。
`build_workbook.py` はこの仕様を実装し、`inspect_workbook.py` が機械検査する。

## 1. カラーコード（フォント色。背景色ではない）

| セル種別 | フォント | Hex |
|---|---|---|
| ハードコード入力 | 青 | `0000FF` |
| 同一シート内数式 | 黒 | `000000` |
| 他シートへの直接リンク（単一参照のみの数式。複数参照が混ざる数式は黒） | 緑 | `008000` |
| ラベル・見出し | 黒 | `000000` |
| 注記・備考・出典 | グレー斜体 | `808080` |

- 青は入力の**フォント色**。入力を背景色で表現しない。
- 背景フィルの許容は2箇所のみ: シートタイトル帯（ネイビー `1F3864`×白太字）と
  チェック行のエラー時条件付き書式（赤字）。それ以外のフィル禁止。
- 複数参照が混ざる数式は黒。純粋な直接リンク（`='Sheet'!C5` 単体）のみ緑。

## 2. レイアウト

- A列は幅2のスペーサー。値・書式を置かない。
- 1行目は空ける。全コンテンツは2行目・B列起点。シートタイトルはB2（帯はB..最終列に個別フィル、結合しない）。
- **単位専用列（E列）**を持つ（FAST: 単位の問いを常に先回りする）。行ラベルに単位を混ぜない。
- エラー級チェックを持つシートのヘッダー右上に**シート内チェックフラグ**（自シートのエラー級チェック行の自己集計数式。要確認時は赤字CF）。総合判定はサマリーのみが持つ。下流シートへの逆参照（表示用でも）は禁止。
- **入力ゾーン規則**: 青字入力セルは前提条件シート（＋バリュエーションのコンプ・取引表）のみ。それ以外のシートの青字数値は defect。コンプ・類似取引の市場観測データは使用場所（バリュエーションシート）上の入力ブロックが正位置（IB実務のコンプページ慣行。事業前提と混ぜない）。
- B3に単位・符号規約の注記（例: 単位：百万円（単価・給与・件数は明記）。費用は正表示）。
- 階層はGoogle Sheets 20px相当（xlsx幅2.14）のインデント列で表現する。レベル0=B列、レベル1=C列、レベル2=D列にラベルを書き、階層が深くなるほど右の列に書き出す。ネイティブインデント（alignment.indent）・先頭スペースは禁止。
- 期間ヘッダー行は全シート同一行・同一列（C列起点、FY表記、太字、下罫線）。
- 最終期間列の右隣に「備考」列（イタリックグレー、幅44）。Source/driver列は設けない。
- 列幅: A=2、B/C=2.14（インデント列）、D=40（ラベル本体）、E=9（単位列）、期間列=11.5、備考=120（印刷範囲の終端）。全シート共通。期間はF列起点。
- ラベルは表示幅がラベル列に収まる長さにする（単位列へはみ出す長さは defect）。
- 行の高さは全行Excelデフォルト（`row_dimensions`の高さ指定禁止）。
- 結合セル禁止。グリッド線は非表示（`sheet_view.showGridLines = False`）。
- セクション間は空行1行。セクション見出しの下罫線・小計/合計行の罫線は備考列まで引く。
- 全期間共通の入力は全期間列に同値を複製した青字行とし、下流の数式は同列セルを参照する（期間整合しないシート＝資本政策・バリュエーション等からは先頭期間列への絶対参照）。単一セル入力＋絶対参照で済ませない。例外はスイッチ類（アクティブケース・認識基準等 `scope: single`）のみ：単一セル＋絶対参照とし、期別に値が割れる編集事故を構造的に排除する。

## 3. 数値書式（Excel format code）

| 種別 | コード |
|---|---|
| 金額（百万円表示、生円値を格納） | `#,##0,,_);(#,##0,,);"-"_)` |
| 金額（円表示: 単価・給与等） | `#,##0_);(#,##0);"-"_)` |
| 比率（斜体） | `0.0%_);(0.0%)` |
| 件数・人数 | `#,##0_);(#,##0);"-"_)` |
| 倍率 | `0.0x_);(0.0x)` |

- セル値は常に生の円（またはネイティブ単位）。百万円表示は書式のスケーリング
  （`,,`）で行い、値を事前に割らない。
- 負数は()表示、ゼロは "-" 表示。同一表内で小数桁を揃える。
- 比率行はフォント斜体（BIWS流儀）。

## 4. タイポグラフィ・罫線・強調

- 全セル Arial 10。サイズ・フォントを混在させない。
- 太字: シートタイトル、セクション見出し、期間ヘッダー、主要合計行のみ。
- 小計・合計行は上細罫線。最終合計（売上高計・当期純利益・期末現金等）は上細＋下二重罫線。水平罫線の左端はインデントに関わらず**B列起点**で揃え、ミニヘッダーを含め途中の列で途切れさせない（右端はデータ末尾または備考列）。
- 費用は正表示とし、利益行の数式で控除する（符号規約はB2注記に明記）。
- 罫線・太字・配置で強調し、フィルに頼らない。

## 5. 数式規律

- すべての導出値は生きたExcel数式。計算結果のハードコード禁止。
- 数式内への定数埋め込み禁止（すべての前提はAssumptionsの青字セルに置き参照する）。
- OFFSET / INDIRECT / NOW / TODAY / RAND等の揮発関数禁止。循環参照禁止。
- リンクのリンク（数珠つなぎ）を避け、原本セルを直接参照する。
- 同一行の期間方向は同一数式のコピーで構成する。
- チェック行（タイアウト・現金非負）は差分を数式で計算し、条件付き書式で
  異常時のみ赤字にする。

## 6. シート・印刷

- シートタイトル→単位注記→（空行）→期間ヘッダー→本体、の順を全シートで統一。
- 同じ期間は全シートで同じ列に置く（time rulerの一貫性）。
- 印刷: 横向き、幅1ページ、印刷範囲を設定し、1〜5行目（タイトル・期間ヘッダー）を繰り返しタイトル行に指定する。PDF出力で切れ・重なりがないこと。

- 備考列は印刷範囲の終端に置き、空のバッファ列を印刷範囲に含めない（列幅120＝全角60字を上限とし、超える注記は分割・短縮する）。
- 期首境界値は稼働数・人員・現金・繰越欠損金に加え、売掛金・買掛金・前受金も入力行で持つ（数式内の暗黙ゼロは defect）。
- 全期間共通の入力（KPIしきい値等）を参照する評価式は、評価対象と同じ期間列のセルを参照する（F列固定参照は編集事故ベクトル）。
- 前受金の期首入力行は、前受金をモデル化する事業（前払・年間一括課金等）でのみ必須。月次後払いで前受が発生しない事業では、その旨を運転資本の注記に明記する。
- シナリオスイッチ等の CHOOSE は `CHOOSE(MEDIAN(1, スイッチ, ケース数), …)` でクランプする（範囲外値で #VALUE! が伝播し、総合判定自体が計算不能になる自壊を防ぐ）。
- 期間列幅は11.5を基本とし、イベント列を持つシート（資本政策等）は見出しが収まる幅まで拡張してよい（ヘッダー切れは defect）。
