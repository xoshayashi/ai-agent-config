# シート構築プレイブック④：資本政策（Cap Table）

対象: IBフォーマット年次収支計画ワークブック内の「資本政策」シート。
出典はWeb調査（2026-07）。全行を青字入力＋数式で駆動する。

## 全体方針
- **列軸はラウンドイベント方式**（設立時→シリーズA前→シリーズA後→予備）。FYタイムルーラーは使わない。資本政策は時点スナップショットのロールフォワードであり、日本の実務標準もラウンド×株主マトリクス（smartround: https://jp.smartround.com/public/articles/fundraising/fp3）。FYとの接続は「実施年度」行1行で持つ。
- 株数は**完全希薄化（FD）ベース**で一本化し、SOプールを潜在株式行として常に分母に含める。株数が主、%は導出行。%入力→株数逆算の構造は禁止。

## 1. ラウンド前提ブロック
| 行 | 属性 | 内容 |
|---|---|---|
| 実施年度 | 青 | 資金繰りシートの調達計上年度と一致 |
| 調達額 | 緑 | =資金繰りの調達行（原本参照。再入力しない） |
| ポストマネー評価額 | 青 | 交渉前提として直接入力 |
| プレマネー評価額 | 黒 | =ポスト−調達額 |
| ラウンド前FD株数 | 黒 | =直前イベント列のFD合計（プール拡大後） |
| 株価 | 黒 | =プレ÷ラウンド前FD株数 |
| 新規発行株数 | 黒 | =ROUND(調達額÷株価, 0) |

- **循環回避**: ポスト入力→プレ差引→株価=プレ÷既存株数（新規株数に依存しない）。「ポスト=株価×発行後株数」を定義式にすると循環するため整合チェックに回す。
- 端数: 新規株数の整数丸めによる1株価未満の差はチェックの許容誤差とする。

## 2. 株主構成ロールフォワード
- 列: イベントごとに〔増減株数／株数残高／持分%FD〕。イベント: 設立時／シリーズA前（プール拡大後）／シリーズA後／予備ラウンド。
- 行: 創業者（複数行可）→創業者計→SOプール（潜在株式）→ラウンド別投資家→FD合計→持分%合計。
- 残高=前イベント残高+当イベント増減。繰越はすべて数式。
- **SOプールはシリーズA「前」列で拡大**（プレマネーの分母に入り、希薄化を創業者側が負担する調達実務＝オプションプール・シャッフル。Startup Law & Finance: https://kitanolawfinance.jp/termsheet/optionpool/）。プール株数は青入力（目標%逆算は循環を生むため不採用）、ラウンド後比率10–15%帯をチェックで検証。

## 3. 希薄化・整合チェック
| チェック | 目安・出典 |
|---|---|
| 持分合計100%（各イベント列） | ABS(合計−100%)<0.01% |
| 調達額タイアウト | 株価×新規株数≒資金繰りの調達額（1株未満誤差許容） |
| ポスト整合 | 株価×ラウンド後FD≒ポスト入力（0.5%以内、プール拡大分のずれは注記） |
| 創業者持分帯 | シード後80–90%、シリーズA後60–75%、IPO時30–50%。代表1/3超で特別決議拒否権（Gazelle Capital: https://gazellecapital.vc/sttv-media/shareholding-ratio/） |
| ラウンド希薄化率 | 10–25%目安、シリーズA約15–20%が近年水準。米国参考: Carta中央値 Seed19.5%・A18%（https://carta.com/data/dilution-q1-2024/） |
| SOプール比率 | 10–15%。上場審査実務は発行済の約10%上限目安（GVA: https://corporate.ai-con.lawyer/articles/stock-option/6） |
| 1株あたり価値推移 | 前イベント比下落なら「ダウンラウンド」表示 |

帯（下限/上限）は前提条件シートの青入力とし、評価は数式（IF）で表示する。

## 4. 注記（シート最下部）
- 優先株式条件: 日本は残余財産優先分配1x・参加型が大勢（Coral Capital調査約97%）だが1x非参加型が増加（スタートアップ協会: https://note.com/startup_kyokai/n/n42f1fe610764）
- J-KISS等がある場合は「転換前」注記＋キャップ/ディスカウント（通常20%）適用後の想定転換株数を参考行で（Coral Capital: https://coralcap.co/2019/12/seed-finance-05/）
- 免責（グレー小字必須）:「本シートは指示的な資本政策シミュレーションであり、最終条件はタームシートおよび投資契約による。」

## 複数ラウンドへの拡張
予備列に値を入れるだけで次ラウンドを追加できる構造を保つ。複数ラウンドを本設計に
昇格させる場合は、ラウンドごとに（実施年度・調達額・ポストマネー）の3入力を持ち、
各ラウンドの希薄化率10-25%帯チェックとラウンド間の株価上昇（ダウンラウンド検知）を
チェック行に含める。バリュエーションシートの投資家リターンはラウンド別MOIC/IRRに
分解できるが、収支計画の範囲では直近ラウンドの投資家リターンで十分なことが多い。

## 品質チェックリスト
1. 全イベント列で持分%合計=100%（FD）
2. 調達額が資金繰りと数式リンクでタイ（再入力なし）
3. 株価×新規株数≒調達額、株価×後FD≒ポストの両整合
4. 創業者持分がステージ目安帯内、逸脱時に「要確認」表示
5. SOプールがラウンド前拡大・後比率10–15%帯
6. %行がすべて導出。青/黒/緑の色規約遵守
7. 予備ラウンド列が0埋めで存在し、値を入れるだけで拡張可能
8. 免責・優先条件・J-KISS備考の記載

## よくある失敗
①ポスト=株価×発行後株数の循環定義 ②%だけ管理し株数を持たない ③SOプール後付けで投資家側も希薄化し紛糾 ④調達額の再入力による資金繰りとの乖離 ⑤端数処理放置でチェック常時FALSE ⑥優先分配無視の創業者手取り過大表示
