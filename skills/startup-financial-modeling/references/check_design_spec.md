# チェック設計仕様（独立経路の原則）

このスキルで最も高くついた失敗は、**チェックが存在するのに実効性がない**ことだった。
IB/VC/監査の3ジャッジは毎回「変異テスト（モデルをわざと壊してチェックが鳴るか）」
を仕掛けてきて、そのたびに同じ構造の欠陥が見つかった。以下は再発防止の契約である。

## 原則1: エラー級チェックは「独立経路」で再導出する

**恒等式は禁止。** 両辺が同じセル・同じ経路を読む式は、代数的に必ずゼロになり、
何も検証していない。チェックは、検証対象とは**別の経路**で同じ値を再導出し、
突き合わせる。

| 検証したいもの | ダメな式（恒等式） | 正しい式（独立経路） |
|---|---|---|
| EBITDA | EBITDA −(NI＋税＋償却) ※PLの定義そのもの | EBITDA −（個別費目行から再導出した売上−費用）※合計行を経由しない |
| 当期純利益 | NI −(EBITDA−償却−税) ※PL行を読む | NI −(EBITDA − **原本ドライバー**の償却・除却損・利息 − 税) |
| 法人税等 | 税 − 課税所得×税率 ※課税所得を与件にする | 税 − MAX(0, 税引前 − MIN(期首欠損金, 税引前×控除限度率))×税率 |
| 現金増減 | （定義式の再掲） | 現金増減 −(NI＋償却＋除却損＋運転資本増減−設備投資＋調達) ※すべて原本参照 |
| 期末現金 | 期首＝前期末 ※再帰の自己言及 | 期末現金 −(期初現金＋**累計**現金増減) ※累計経路 |
| 資産簿価 | 期末簿価 −(期首簿価＋取得−償却−除却簿価) ※コークスクリューの定義式の再掲＝**恒等式** | 期末簿価 −(期首取得原価−期首累計償却＋**累計**取得−**累計**償却−**累計**除却簿価) ※累計経路 |
| 有利子負債 | 期末残高 −(期首＋調達−返済) ※同上 | 期末残高 −(期首＋**累計**調達−**累計**返済) ※累計経路 |
| FD株数 | 持分合計＝100% ※構造上必ず成立 | 期末FD株数 −(前イベント＋新規発行)、株価×FD株数 ＝ ポストマネー |
| 人員 | 期末＝期首＋純増 ※純増が差分定義なら恒真 | 期中平均人員 −(期首人員＋期末人員)÷2 |

**判定法**: そのチェックが恒真式かどうかは、代数展開すれば分かる。迷ったら
`scripts/mutation_test.py` を回す。配線を落として財務が動いたのに総合判定が
OKなら、そのチェックは偽の保証である。

## 原則2: 境界値（期首残高）を数式に埋め込まない

期首の稼働数・簿価・累計償却・負債残高・人員・売掛買掛・繰越欠損金は、
**すべて前提条件の入力行**として持つ。`prev(x, 0)` のリテラルゼロは defect。
既存事業に使った瞬間に初年度の計算が静かに消え、しかもロール整合チェックが
両辺で同じゼロを読むため沈黙する。

`scripts/boundary_test.py` が、期首行に非ゼロを投入して自壊しないことを検証する。

## 原則3: スイッチは定義域外で自壊させない

`CHOOSE(スイッチ, …)` は範囲外値で #VALUE! を撒き、**総合判定セル自体が計算不能**
になる。ガードのつもりのチェック行が、発火時に判定不能になっては意味がない。
必ず `CHOOSE(MEDIAN(1, スイッチ, ケース数), …)` でクランプし、範囲整合チェック
（エラー級）で検知する。

総合判定は `SUM(IFERROR(各チェック, 1))` で包み、計算エラーを「要確認」として扱う
（#DIV/0! が出ているのに「OK」と表示する状態を禁じる）。

## 原則4: チェックは二層に分ける

- **エラー級**: 配線・恒等式の破れ。全行ゼロで総合判定OK。出荷不可。
- **アラート級**: 事業アウトカム（現金不足・ランウェイ不足・ベンチマーク逸脱・
  ソース記載レンジの超過・非Baseケース選択）。出荷可・要レビュー。

「不都合な事実」はアラート級で**必ず点灯させる**。隠すと採点で必ず見抜かれ、
開示していれば「誠実」と加点される（実際に3ジャッジ全員がそう評価した）。


## 原則5: ロール整合は「累計経路」で組む

コークスクリュー（期末＝期首＋増−減）を、そのままの形で検算しても**恒等式**である。
期末値がその式で定義されている以上、同じ式で再導出しても何も検証しない。
必ず**累計経路**（期首＋Σ増−Σ減）で組む。これは再帰式とは別経路なので、
中間年の1セルが壊れれば差として現れる。

## 原則6: 単一起点の導出行は「入力まで展開して再計算」する（依存錐で機械判定）

売上・原価・人件費・設備投資・償却・利息のような**単一起点の導出行**は、
その行を落としても、行を読んでいる下流の合計もチェックも一緒に動くため、
どんなに巧妙なタイ式を書いても検知できない。

解決は**入力プリミティブまで数式を展開した再計算チェック**（エンジンが
`roles` の各ラインについて自動生成する「独立再計算チェック」）。
展開式は本体行を一切経由しないため、中間行のゼロ化・脱落が必ず差として現れる。

**機械判定の定義（これが唯一の合格基準）**: チェック式を `対象 −（再導出）` に
二分し、両辺の**依存錐（セル単位の先祖集合）**を取る。あるセル X が
*両側の錐*にあれば、そのチェックは X を守っていない（差が相殺されうる）。
*片側の錐だけ*にあれば守っている。

> **帰結**（当期純利益・期末現金・株式価値・現在価値EV・MOIC/IRR・Exit分配・
> FD株数・持分）の先祖である全導出セルが、いずれかのエラー級チェックの
> **対称差**に入っていること。

これを `scripts/tautology_lint.py` が再計算なしで全列検査する。
`roles` に載っていない中間行（PLの営業利益・税引前・繰越欠損金、資金繰りの
売掛・買掛残高、バリュエーションの割引係数・TV・Exit EV、資本政策の株価・
新規発行株数…）も自動的に宇宙に入るので、「チェックを書いた行だけ守られて
いる」という過去の穴は構造的に塞がれる。

**帰結（シンク）の選び方が最重要**: オラクルを当期純利益と期末現金だけに置くと、
バリュエーションと資本政策への変異が全部「不活性」に分類され、**検査したつもりで
丸ごと素通り**する。ここは過去の採点で最も突かれた領域なので、
**投資家に見える最終出力はすべてシンクに入れる**（リンターと変異テストで同一の
シンク定義を使う）。

**保護を与えないチェック（機械が無視する）**:
- **スケール不変**なチェック（`持分合計 − 1` など再導出側が定数）。株数を一律に
  壊しても比率は100%のまま。水準を固定しないので、何も守っていない。
- `対象 − 再導出` の減算形でないチェック（比率形の `(A*B−C)/C` 等）。業務上の
  意味はあるが、機械保証の根拠にはしない。

## 原則7: 許容差はチェックの単位に合わせる（スケール整合）

チェックが独立経路でも、**許容差の単位がズレていれば沈黙する**。
実例: Exit株式価値・MOIC・IRR を1本にまとめた再計算チェックを **%単位**で書き、
許容差は既定の **0.5（円スケール）**のままだった。結果、適用倍率行を落として
IRRが5pt動いても `|0.05| < 0.5` で **総合判定OK**（盲点5件）。

- **エラー級チェックは原則として金額（百万円）・株数で書く**。金額に落とせるなら落とす。
- %・倍・係数で書かざるを得ない場合は、**許容差を明示**する（0.005以下）。
  エンジンは `Sheet.check()` でこれを強制し、違反はビルド時に `ModelError`。
- 同じ配線を金額と%の両方で守れるなら両方置く（金額側が本命、%側は補助）。

**この欠陥は静的リンターでは見つからない**（経路は独立しているため）。
`mutation_test.py` が実際に壊して初めて出る。だから二層で回す。

## 原則8: オラクルは「印刷される面」と一致させる（ハーネス自身の偽の保証）

チェックが独立経路でも、**変異テストのオラクル（帰結）が納品面を覆っていなければ、
ハーネス自体が偽の保証**になる。

実例: オラクルを当期純利益・期末現金・株式価値・MOIC等に限った結果、
- **DCF内訳表のΣPV（FCF・中位）**（EVは別の `NPV()` 行から作られるため、
  どこにも流れない「死に行」だった）
- **感応度グリッド**（25セル中、エラー級で結ばれていたのは中位×中位の1セルだけ）

を壊しても、どの帰結も動かないので「**不活性**」に分類され、**検査対象から除外**された。
ハーネスは「盲点0」でPASSしたが、投資家が読む数字は壊れたままだった（監査ジャッジが実証）。

対策:
- **印刷面に出る数値行は、帰結に流れるか、エラー級チェックで結ぶかのどちらか**にする。
  どちらでもない行は「死に行」＝二重計算であり、**行ごと消すか、タイを張る**。
- `mutation_test.py` は**不活性行を必ず一覧出力する**。PASSしても、そのリストに
  意思決定に効くはずの行が並んでいないかを毎回目で見る。
- 同型の失敗が `boundary_test.py` にもあった: 期首入力を**全列に投入**していたため、
  期首残高を**同列参照**していた欠陥（既存借入のある会社に転用した瞬間に返済
  スケジュールが壊れる）を素通りさせた。→ 期首はF列単独投入に変更し、エンジンは
  期首入力への参照を**常に `$F$` 固定でコンパイル**する。

## 契約（唯一の合格基準・モデル非依存）

> **印刷範囲に出る導出セルは、すべてエラー級チェックで守られていること。**
> 壊れたら必ず総合判定が鳴る。鳴らない数字を投資家に見せない。

検査対象は `scripts/model_surface.py` が**構造だけ**で決める。ラベルの列挙も、
「帰結（シンク）」の列挙も、対象シートの列挙もしない（＝モデルごとの個別調整が要らない）。

| | 定義（構造） |
|---|---|
| 検査対象 | 各シートの**印刷範囲内**の数式セル |
| 除外: 入力シート | 前提条件（入力値の誤りはチェックでは原理的に検知できない＝境界） |
| 除外: 統制行 | サマリーの集約が読む行・総合判定・警告件数・ソース照合の差異行 |
| 除外: 文字列行 | 文字列を返す式（「期間内なし」等。算術タイで守れない） |
| 印刷範囲外 | チェック明細（監査証跡）。意思決定面を統制ログで埋めないため |

この定義から、**表示専用行も自動的に検査対象になる**（マージン・成長率・ランウェイ・
コンプ中央値・サマリーの見出しミラー）。

**エンジンは印刷面の全導出行に「行タイ」を自動生成する**（`add_row_ties`）:

    行タイ ＝ (印刷面の各セルの合計) −（各セルの式を再計算した合計）

- 下流に効く行を独立再計算チェック（入力まで展開）で守るだけでは足りない。
  **その年の数量がゼロなら、行を壊しても下流が動かず、どのタイも鳴らない**
  （ACTで実際に発生: Regulated月額ARPA・月間稼働分数）。
- 行が別の行を読むと同じ式の中で差が相殺されるので、**依存の深さで層に分け、
  層ごとに1本**出す（浅い層のセルは浅い層のチェックが守る）。深さの打ち切りは禁止——
  打ち切ると深い行が同じ層に潰れて相殺する。
- 展開しきれず残ったドライバー（深いコークスクリュー）には、**それ自身の独立再計算
  チェック**を張って閉包を取る（`add_derive_checks` の残余閉包）。式の爆発を避けつつ、
  どの中間行も必ずどこかのチェックの片側だけに現れる状態を保つ。
- **チェック本体はシート末尾・印刷範囲外**に置く（意思決定面を統制ログで埋めない）。

## 原則9: ハーネスの「対象外」は必ず構造で決める（ラベルで決めない）

3ジャッジがそろって突いた最後の穴は、**ハーネスの対象選定**だった。

- **ラベルで除外していた**: 変異テストは「（参考）」を含む行を対象外にしていた。
  ところが `（参考）アンレバードFCF（FCFF）` は**DCFが参照する唯一のFCF行**で、
  ゼロ化するとDCF株式価値が+72%動くのに総合判定はOKのままだった。
  → チェック行は**サマリーが集約している行**として構造的に特定する。
  ラベル（「整合」「（参考）」）で判断してはならない。**ラベルは名前であって型ではない。**
  そして「参考」と名付けた行が実は載っている行なら、**名前のほうを直す**。
- **中央列しか壊していなかった**: `MIN/MAX/IF` は摂動を吸収する。
  `採用Exit EV = MAX(類似上場, 類似取引)` の非採用側をゼロにしても MAX は動かず、
  中央列だけの変異では露見しない（高位列でだけ露見する）。
  → 変異は**全期間列**の各セルで行う。静的リンターは MIN/MAX の飽和を見抜けないため、
  ここは動的ゲートでしか埋まらない。
- **KPIシートを対象外にしていた**: 感応度グリッド9セル中、タイが張ってあったのは
  中位×中位の1セルだけだった。→ グリッドは**全列を入力から再導出**して結ぶ。

## 原則10: 「対象の式を写した再導出」は設計ミスを祝福する。ヘッドラインは横断不変条件で守る

もっとも高くついた見落とし。バリュエーションの手法②③が

- 割引率を**中位に焼き込んで**いた（低位列も高位列も中位で割引き）
- **計画期間のFCF現在価値を負担していなかった**（①だけが負担）

にもかかわらず、行タイも「■必達 3手法の値の一致確認」も**すべて PASS していた**。
理由は単純で、再導出側が**対象セルの式をそのまま書き写していた**から。

```
F50: =F49/(1+G20)^n*(1-DLOM)          ← G20（中位）を焼き込んだ本体
検算: =(F50)-((F49/(1+G20)^n*(1-DLOM)))  ← 同じ間違いを写した「独立経路」
```

**再導出が対象と同じ誤解から書かれているなら、それは独立経路ではない。**
入力プリミティブまで展開してもこれは起きる（プリミティブの**選び方**が同じ間違い
だから）。静的リンターも動的変異も PASS する。**依存錐は「経路が違うか」しか見ず、
「基準が同じか」は見ない。**

対策は、式の写しではなく**関係式**を問うこと:

> 各手法の「株式価値 − 自分のExitアンカーの現在価値」は、3手法で一致するはずだ。
> （残差＝計画期間FCFの現在価値＋現金−有利子負債＝共通ブリッジ）

```
■必達 手法②が手法①と同じブリッジか: =(手法②−②のExit現在価値)-(手法①−①のPV(TV))
```

これは②の式を一切写していない。②がブリッジを負担していなければ、その差
（本件では53十億円）がそのまま残って必ず鳴る。

**規則（汎用）**

1. **同じ量を別の道で出す行が複数あるなら、道ではなく「差が0であること」を問う。**
   フットボールフィールド、三表整合、期首=前期末、分配合計=総額——すべてこの形。
2. **意思決定に使う値（サマリーに出る結論・採用レンジ・MOIC）は、行タイ（値の破損
   検知）だけでは不足**。必ず横断不変条件を1本持たせる。
3. **自動生成の行タイの役割を誤解しない。** あれは「セルが壊れたら鳴る」ための網
   （mutation coverage）であって、**式の設計が正しいことの証明ではない**。

## 原則11: 印刷面の青字入力は、すべて参照されていなければならない（孤児セル禁止）

期初現金は `$F$` 固定で参照されるのに、**値は F〜J の全列に青字で複製**されていた。
G〜J は「入力」として紙面に出ながら誰にも読まれない。監査ジャッジが実演したとおり、
**J列の期初現金を990億円に書き換えても総合判定は OK のまま**である。

チェックの穴ではない（そのセルは何も動かさないので、そもそも壊れようがない）。
**読み手を欺く紙面**が問題である。編集できるように見えるセルが、編集しても何も
起こさない——これは「この数字を変えれば計画が変わる」という約束の反故。

規則:

- **印刷面の青字セルは、必ずどこかの数式から参照されること**（機械ゲート）。
- 全期間共通の定数（税率・要求収益率・しきい値）→ 2年目以降は `=$F$r` の**数式**にする。
  見た目は同じだが入力ではなくなるので、行タイと変異テストの網に入る（壊せば鳴る）。
- 期首・期初の境界値 → 2年目以降は**空**。期初現金が5年並ぶのは意味的に誤り。
- 中央値算定外の参考コンプのように「置くが使わない」数値は、
  **セルにせず備考に書く**か、参照される行（外れ値込みの中央値など）を足して生かす。

## 原則12: ベンチマーク逸脱は、必ず意思決定面に出す

KPIシートで赤字「要説明」と判定された5項目（営業効率17.2倍・S&M比率5.9%・
資金燃焼倍率16.5倍・償却後粗利率28.3%・ランウェイ4.5ヶ月）が、**サマリーの警告に
1件も出ていなかった**。この計画で最も不都合な数字が、1〜2ページ目から見えない。

評価列（良好／水準内／要説明）は**構造**である。そこから機械的にアラート級チェックを
生成し、サマリーの警告レジスタに配線する。「KPIシートを見れば分かる」は逃げ道。

## 原則13: チェックの「和」に単位を混ぜない（桁落ちによる偽の保証）

原則7（許容差は単位に合わせる）は、**チェック本体の構成にも効く**。

行タイを1本にまとめていたため、1つの和に

- 金額（期初現金 10億円、調達 80億円…）
- 比率（Gordon永久成長率 0.025、許容誤差 0.01…）

が同居していた。107セル・合計 **7.85兆円**。ここで比率セルを1つ壊しても、
差 0.025 は合計に対して相対 **3.2e-15** ——**float64 の精度の下に沈んで消える**。
静的リンターは「依存錐が非対称だから守られている」と言い、**動的変異テストだけが
盲点として実証した**（ATOM 12件・ACT 38件）。

規則:

- **タイは数値書式（＝単位）ごとに分ける。** 金額のタイ、%のタイ、倍のタイ、
  整数のタイ。同じ書式なら同じスケールなので、破損が精度に埋もれない。
- **許容差は最小（±0.0001）にする。** タイは左右が同じ順序で同じ項を足すので
  差は厳密に 0 になる。許容差は**丸め誤差の閾値**であって「見逃してよい幅」ではない。
  小数クラスに ±0.5 を使ったため、値 0.357 の「導入密度余力」をゼロ化しても
  差 0.357 が許容差に収まり、**単位を分けた後もなお盲点が1件残った**
  （2巡目の変異テストが検出）。
- これは「片方のゲートだけでは足りない」ことの最良の実例。
  **静的（設計）と動的（実挙動）は違うものを見ている。**
- そして「1回直して終わり」でもない。**盲点が0になるまで変異テストを回し切る**
  （12件 → 1件 → 0件と、2段階の別原因が順に出た）。

## 原則14: アラートの走査範囲は、ラベルの主張と一致させる（「いずれか」と言うなら全セルを見る）

3ジャッジ全員が critical として挙げた欠陥。

```
□要説明 いずれかの手法で株式価値が0以下
   式: =IF(MIN(G92,G93,G94)<=0,1,0)      ← 中位列しか見ていない
```

低位列では手法① ▲11,481百万円、手法② ▲21,327百万円 と、**同じPDFに負の株式価値が
印刷されている**。それでもサマリーの警告レジスタは「—（該当なし）」と印字する。
**沈黙より悪い。読み手の目の前にある事実を、モデルが積極的に否定している。**

なぜ機械ゲートを素通りしたか——**変異テストのオラクルが「総合判定（エラー級）」
だけを見ており、アラート層を一度も試験していなかった**。カバレッジ100%・盲点0を
達成しても、それはエラー級の話でしかない。

規則:

- **ラベルが「いずれか」「全期間」「どこかで」と言うなら、対象の全セルを走査する。**
  `SUMPRODUCT(--(範囲<=0))` の形にする。単一セル参照で全域を名乗らない。
- **スカラーのチェック（1列しか値を持たない）は、年・ケースを名乗らない。**
  F列に置いてあるだけなのに列ヘッダー（「低位」「FY2026」）を印字すると、
  「低位ケースだけの話」と誤読される（実際はBaseで鳴っている）。
  → 構造で判定する（`formula(col, 1) is None` ならスカラー → 表示は「計画全体」）。
- **アラート層にも実演（試験）を要求する。** エラー級の変異テストは
  アラートの正しさを何も証明しない。

## 原則15: 逆算の主張は、備考を禁じても見出しに逃げる

原則10で「導出行の備考が検証を名乗る」ことを禁じたら、同じ主張が**セクション見出し**
に移って生き延びた:

```
見出し: 「ソースの看板主張（18か月回収）を、モデルの数字で検証する行」
実体:   製造原価が「18ヶ月回収からの逆算」【逆算】タグ付き
        → 回収期間 = 原価 ÷ 月商 は恒等的に 18.0 にしかならない
```

**前提から解いた数字で、前提を検証している。** 逃げ道は語ではなく構造で塞ぐ:
**印刷面のあらゆるテキスト**（ラベル・見出し・説明行）で「検証」「一致」を
名乗らせない。名乗ってよいのはチェック行とその集約行だけ（機械ゲート）。

## 原則16: 「1本直した」は「直った」ではない。同型を機械で掃く

原則14（アラートの走査範囲）を作り、該当する警告を**1本だけ**直した。
次の巡で、IBジャッジが**残り3本**を実演した:

```
F119 = IF(G82 < 目標, 1, 0)              ← MOIC目標未達。中位列のみ
F120 = IF(ポストマネー > G77, 1, 0)        ← 評価額超過。中位列のみ
F121 = IF(OR(G98>3, MIN(G93:G95)<=0),…)  ← 3手法の食い違い。中位列のみ
```

しかも**低位ケースは構造上いちばん先に壊れる側**（株式価値が最小 → 評価額比が最大、
MOICが最小）。「中位は通るが低位が落ちる」は例外ではなく**自然な並び順**であり、
偽陰性は最も起きやすい方向に開いていた。

規則:

- **原則を作ったら、同型を機械ゲートで掃く。** 手で1本ずつ直すと必ず取りこぼす。
- 実装: ケース列（低位/中位/高位）を持つシートで、ケース依存の行を参照している
  アラートが**F列・H列を一度も参照していない**なら FAIL（`inspect_workbook.py`）。
- 同じことが**許容差**にも起きた: 原則13（単位ごとにタイを分ける）を行タイに適用した
  が、**サマリーの転記タイには適用し忘れた**。10単位36セルが1本の和（4.4e11）に入り、
  MOICを許容差（±0.0001）超の 1.4e-4 で改ざんしても桁落ちで消えた。
  **p.1の見出し数値を守る唯一のチェックが、自ら宣言した許容差で機能していなかった。**
- **ゲートを足したら、退行変異でゲート自身を試験する。** 効かないゲートは偽の保証。

## 原則17: 軸の語彙を混ぜない（列の名前を、ケースの名前として使わない）

バリュエーションの列は「低位/中位/高位」（＝倍率と要求利回りの保守〜強気）。
シナリオは「Base/Upside/Downside」（＝事業前提の強弱）。**これは別の軸である。**

混ぜた結果:

- サマリーに「（参考）Downsideケースの回収倍率」を新設し、
  バリュエーションの**低位列**MOIC（1.85倍）を引いていた。Downsideではない。
- バリュエーションには売上比の線形按分による別の「Downsideケースの回収倍率」
  （3.19倍）があった。エクイティ・ブリッジの純負債控除が固定額なので按分は成立しない。
- 実際にDownsideを回すと 3.59倍。**同じ名前の数字が3つあり、うち2つが同一PDFに印刷。**

規則:

- 印刷面のラベルに**ケース名（Base/Upside/Downside）を使ってよいのはシナリオ軸の行だけ**。
- 列軸の話は列名（低位/中位/高位）で言う。警告の位置表示も同じ。
- **成立しない近似には名前を付けない。** 近似に事実の名前を付けると、読み手は事実として読む。

## 原則18: 走査側を直しても、報告側が嘘をつく（偽陰性は偽陽性に置き換わるだけ）

原則14（アラートの走査範囲）を機械ゲート化し、ケース依存の警告4本すべてを
全ケース走査に直した。**走査は正しくなった。** それでも3ジャッジ全員が critical を出した:

```
□要説明 投資家の回収倍率が目標を下回る   → 「全ケース」と印字
   実際: 低位1.85・中位6.38は未達だが【高位12.42は目標10倍を超過】
□要説明 いずれかの手法で株式価値が0以下  → 「全ケース」と印字
   実際: 負なのは【低位列の2セルのみ】。中位・高位は9セル全て正
```

件数を数える `SUMPRODUCT` は全列を走査していた。だが**位置ラベルが定数文字列**
（`TRIM("全ケース")`）で、走査結果を一切読んでいなかった。
同じPDFのp.27が「目標超過」と印字しているのに、p.3が「全ケース」と否定する。
**中位列しか見ていなかった偽陰性が、全ケースと言い張る偽陽性に置き換わっただけ。**

規則:

- **チェックは「件数」ではなく「ケース列ごとの 0/1 ベクトル」で持つ。**
  そうすれば位置表示は、実際に点灯した列の見出し（低位/中位/高位、FY2027…）を
  連結するだけで自動的に事実と一致する。集計は集約行の仕事。
- **定数文字列を位置表示に渡さない。** 「全ケース」「計画全体」は、
  本当にスカラー（1列しか値を持たない）なチェックにだけ許される。
- 教訓の一般形: **走査・判定・報告は別の層であり、片方を直しても他方は直らない。**
  修正のたびに「その事実が、紙面のどこに、どう出るか」まで追い切る。

## 原則19: 切り詰めで意味が変わるラベルは、機械では見つからない

位置表示のセル値は正しく `FY2027 FY2028 FY2029 FY2030`（4年）だった。
だが右寄せのまま列幅（11.5）を超えたため、PDFには `/2029 FY2030` と印字された。
**4年の問題が2年に見え、最も逼迫した年（FY2027 11.5ヶ月・FY2028 6.5ヶ月）が消えた。**
同型で「低位〜高位の全ケース」が「高位の全ケース」と切れ、**事実と逆の意味**になった。

- 文字列セルは**左寄せ**にして、空いている右隣へ流す（結合セルはIB作法で禁止）。
- 印刷面の文字列は、描画幅が列幅を超えるなら、右隣が空でない限り FAIL（機械ゲート）。
- **セルの値が正しいことは、紙面が正しいことを意味しない。** 最後は必ずPDFを見る。

## 二層の機械ゲート（役割分担）

| | `tautology_lint.py`（静的） | `mutation_test.py`（動的） |
|---|---|---|
| やること | 依存錐の対称差で「守られていないセル」を列挙 | 実際にセルをゼロ化して再計算し、総合判定が鳴るか |
| 速度 | 数秒（LibreOffice不要） | 十数分（**全行×全列・並列8**が既定） |
| 網羅 | 全期間列（線形の依存関係のみ） | 全期間列（**MIN/MAX/IFの飽和も含む実挙動**） |
| 見るもの | 設計の欠陥（偽の保証） | 実挙動（式の書き間違い・飽和・不活性行） |

**静的が「守られている」と言っても、動的は嘘だと言うことがある**（MAXの非採用側を
壊しても差が出ない、など）。逆に動的は不活性行を検査対象から落とすため、
静的の宇宙（帰結の先祖）で穴を塞ぐ。**両方を通して初めて出荷できる。**

静的で落ちたら設計を直す。動的で落ちたら実装を直す。**両方を verify.sh のゲート
に置く**（片方だけでは、過去に見逃した欠陥クラスがどちらも残る）。

## ハーネスの限界（免責。必ず読むこと）

両ゲートが PASS でも、以下は**検証されていない**:

- **前提条件の入力値そのものの誤り**（単価を10倍にした等）。チェックでは原理的に
  検知できない。ソース照合ブロックと人間のレビューで担保する。
- **どの帰結も動かさない表示専用行**: KPIのベンチマーク評価ラベル、感応度グリッドの
  周辺セル、参考注記行。→ `boundary_test.py`・`scenario_sweep.py`・採点で担保する。
- **チェック式そのものの意味の誤り**（独立経路だが再導出のロジックが間違っている）。
  リンターは経路の独立性しか見ない。式の意味はレビューで担保する。
- **再導出が対象と同じ誤解から書かれている場合**（原則10）。両ゲートとも PASS する。
  → ヘッドラインには**横断不変条件**（手法間・表間の差が0）を必ず1本置く。
- **ARR/発生主義の分離・償却後開示・ソース忠実性**といったモデリング判断。
  ここは `playbook_capital_intensive.md` とルーブリック採点の領分。

**PASS は「配線が壊れていない」ことの証明であって、「モデルが正しい」ことの証明ではない。**
