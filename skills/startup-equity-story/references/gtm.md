# GTM — 焦点のあったICP設定・GTM戦略・市場への浸透

## 一本のストーリーへの接続

- 直前(市場・機会)から受ける: 「市場は大きく、構造変化が起きている。**したがって**、その痛みが最も深く・最も早く買う一群(=beachhead ICP)がここにいる」。市場スライドのSOMは、この要素のICP定義と同じ母集団でなければならない。
- 直後(財務計画・資金使途)へ渡す: 「このICPに対し、この単価×このチャネルで、CAC・サイクル・CVRが実測できた。**したがって**、資金を投下すれば同じ機械が予測可能に回り、財務計画のトップラインはこのGTMの逆算値そのものである」。
- 接続の検算: 市場章のSOM ≧ GTM章の年次獲得計画の累積、GTM章のfully-loaded CAC×年間新規獲得数 ≒ 財務章の新規獲得対応S&M費(差異10%以内)、GTM章の人員計画 = 財務章の採用計画。3章の数字が突合しない資料は、この要素で必ず落ちる。

## ベスト構成(節立てレベル)

### 1. ICP定義 — 「誰が・いつ買うか」を4軸で特定
- 業種 × 規模(従業員数/売上帯)× 役職(経済的買い手と利用者を分離)× **トリガーイベント**(監査指摘、担当者退職、制度改正、資金調達、システム更改期限 等)の4軸で1枚に書く。
- 「なぜこの一群のペインが最も深いか」を定量で示す: 課題に費やす時間・金額、現状の代替手段のコスト、放置時の損失。
- 反証としての除外定義を添える: 「誰には売らないか」(例: 従業員50名未満は対象外、理由: 決裁者不在でCVRが1/3)。除外の明記は焦点の証明になる。

### 2. Beachhead選定の判定基準 — 360度発散→ペイン深度優先で収束
Moore『Crossing the Chasm』の実務形。**発散→検証→収束**の3段で選定根拠を表で見せる:

**(1) 360度の候補出し(発散)**: 思いついたセグメントから始めない。市場の切り口を全方位で列挙してから候補を作る — 業種 / 企業規模 / 役職・部門 / ユースケース(JTBD)/ 地域 / チャネル・商流 / 規制・トリガーイベント / 技術成熟度(クラウド化済み等)の8軸を最低一巡し、各軸で「ペインが最も深そうな一群」を挙げる。ここで5〜10候補に広げる(3個未満は検証不足のサイン)。

**(2) ペイン検証**: 各候補のペイン深度を一次証拠(顧客の発話、現行代替への支出額、放置損失、支払実績)で検証する。推定のみの候補はスコアに `[estimate]` を明記。

**(3) スコアリング(収束)**: 候補を次の4軸で1〜5点評価する:
- **ペインの深さ(第一基準)**: 問題の経済価値(解決額)。市場の大きさではなく「問題の深刻さが速くキャズムから引き上げる」(Moore)。
- **到達容易性**: リスト化可能か、既存の接点・チャネルがあるか、意思決定者に直接届くか。
- **紹介連鎖・リファレンス性**: 同セグメント内で口コミが伝播するか。pragmatist(実利派)は同業のリファレンスがないと買わない — 口コミの臨界量こそキャズム越えの唯一の手段(Moore)。
- **拡張性(ボウリングピン)**: このセグメントの制覇が次のセグメント攻略を助けるか(顧客・パートナーの重なり)。
- 判定ルール: **ペイン深度が最大の候補を最優先GTMに設定する**(ペイン深度が同点の場合のみ他3軸で決める。他軸の高得点でペイン深度の低さを補って選ばない)。いずれかの軸が1〜2点なら候補ごと棄却(show stopper)。「big enough to matter, small enough to lead」— 12〜24か月でセグメント内シェア30%超を狙えるサイズに絞る。
- Whole product(導入・連携・サポート込みで顧客の課題が完結するか)の欠落領域と、誰が埋めるか(自社/パートナー)を明記。

### 3. ACV×チャネル整合 — 単価とsales motionの相場表
Christoph Janzの「5 animals」(象=$100k+×1,000社、鹿=$10k×10,000社、兎=$1k×100,000社…)を土台に、自社ACVと motion の整合を1枚で示す:

| ACV帯 | 成立するmotion | 経済性の根拠 |
|---|---|---|
| 〜$5k(〜75万円) | セルフサーブ/PLG | 人を介す余裕がない。CVRとバイラルが生命線 |
| $5k〜$15k | PLG+インサイド補助 | 人は「後押し」役 |
| $15k〜$50k | インサイドセールス(SDR+IS) | ISレップ: 年$130-180k費用で年25-40件成約が相場 |
| $50k〜$150k | フィールド寄りハイブリッド | |
| $150k+(2,000万円+) | エンタープライズ直販(named accounts) | フィールドレップ: 年$250-400k費用で年8-15件成約が相場 |

- 検算式を明示: 「レップ費用 ≦ レップ年間成約ARR × 目標CAC比率」が成り立つか。ACVは鹿なのに象並みの営業工数がかかる構造(mismatch)は成長停滞の典型因(Janz)。
- ACVを上げる/下げる計画があるなら、motionの組み替えコスト(人材タイプ交代)まで書く。

### 4. 浸透の数学 — SOM→CAC予算の逆算チェーン
GTMと財務計画をつなぐ検算。次の順に1枚で示す:
1. SOM(ICP該当社数 × ACV)→ 5年の年次獲得社数計画(SOM浸透率で妥当性確認。年5%超の浸透を置くなら根拠必須)
2. 必要新規ARR ÷ 商談勝率 = 必要パイプライン(勝率の逆数がカバレッジ。勝率33%なら3x、20%なら5x。qualified oppの平均勝率は約29%)
3. 必要パイプライン ÷ ステージ転換率 = 必要リード数(相場: MQL→SQL 15-35%、SQL→商談 30-50%、商談→受注 20-35%。自社実測値を使い、相場は妥当性の傍証に)
4. 必要リード数 × リード獲得単価 + 人件費 = 必要S&M予算 → CAC・CAC payback(LTV/CAC 3:1が下限相場)に還元
- このチェーンの各係数に「実測値/相場/仮置き」のラベルを付ける。実測が1つもない逆算は計画ではなく願望。

### 5. 再現性の証明 — 「創業者の魔法」ではなく「機械」
- **Sales Learning Curve**(Leslie & Holloway, HBR 2006): 営業を先に増やさず、まず3〜4名のレップで「レップの限界貢献が黒字化する歩留まり」に達してから増員する。この順序を踏んでいる事実自体が証拠になる。
- 創業者売り→組織売りの移行証拠: 「創業者が関与しないパイプラインから受注が出た比率」を時系列で示す(68%の創業者が初回のこの移行に失敗する — 移行済みは強い差別化)。
- プレイブック化の証拠: ICP定義書・トークスクリプト・オブジェクション対応・POC標準手順が文書化され、新人レップの立ち上がり月数(ramp)が短縮していること。
- 効率指標: Magic Number(純新規ARR×4 ÷ 前Q S&M費)。Seed/Series Aで0.6〜0.9、1.0超が続けば「増員すれば比例して伸びる」の定量証明。CAC payback(月数)、レップ別quota達成率の分布(トップ1人でなく中央値)も添える。

### 6. 拡張戦略の型 — それぞれの証拠要件
- **Land & Expand**: 証拠はNRR。相場: エンタープライズ中央値118%、ミッドマーケット108%、SMB97%。拡張ARRが新規ARRに占める比率(スケール後は5〜6割が相場)と、拡張の再現メカニズム(席数/使用量/追加モジュール)を示す。NRR120%超の上場SaaSはEV/Revenueで約3倍のプレミアム。
- **隣接セグメント**: ボウリングピンの実績1本(beachhead経由で隣接に1社以上入った事例)+ 転用可能な資産(リファレンス、whole product部品)の特定。
- **地理**: 現地固有のGTM前提(チャネル、規制、商習慣)を1か国分は具体化してから語る。「グローバル展開」の一言は減点。
- **複数プロダクト**: 既存顧客のattach rate実績、または顧客からの明示的要望リスト。実績ゼロなら「オプション」として財務計画から外す。
- 優先順位を1つに絞り、他は時期を明示して後置する。同時多方面は焦点欠如と読まれる。

### 7. Latent demand — 追加資金が解放する需要(NFX)
- 「資金がボトルネックで取りこぼしている需要」を具体で示す: 対応しきれない有効リード数、パートナーからの未消化紹介、ウェイトリスト、勝率が立っているのに人手不足で追えない商談。
- 調達額別プラン(例: $3M案と$5M案)で獲得曲線がどう変わるかを提示できる状態にしておく。同じ計画の「延命」ではなく「加速」であること。

### 8. 日本市場特有の論点(該当時)
- 大企業は基幹系を握るSIer・ITコンサル経由で意思決定する比率が高く、地方・中小約360万社は地場ITベンダー・IT商社経由が主流。直販(THE MODEL型)だけでは届かない層を明示し、直販/パートナーの線引きを書く。
- SIerはカスタマイズ工数が薄いSaaS販売への経済的動機が弱い。パートナー戦略を語るなら「パートナーの儲け(導入支援・運用受託・リセールマージン)」の設計を添える。CSをパートナーに委ねる場合の品質担保も論点。
- エンタープライズは稟議・年度予算サイクルが支配的。トリガーイベント(予算期、制度改正)と提案タイミングの整合、稟議を通す数値ビジネスケースの提供を武器として書く。業界団体・ユーザー会経由のリファレンス連鎖は日本のpragmatist攻略で特に効く。

## 合格基準チェックリスト

- [ ] ICPが業種×規模×役職×トリガーイベントの4軸で1文に書ける(「誰には売らないか」も明記)
- [ ] beachhead候補が8軸(業種/規模/役職/ユースケース/地域/チャネル/トリガー/技術成熟度)の360度発散で5個以上挙がり、各候補のペイン深度に一次証拠がある
- [ ] beachhead選定がスコアリング表で示され、ペイン深度を第一基準として最優先GTMが決まっている(到達性・リファレンス連鎖・拡張性は同点時の従属基準)
- [ ] beachheadは「12〜24か月でセグメント内シェア30%超」を狙える粒度に絞られている
- [ ] ACVとsales motionが相場表と整合し、レップ経済性(費用≦成約ARR×CAC比率)の検算が通る
- [ ] SOM→年次獲得→必要パイプライン→必要リード→S&M予算の逆算チェーンが1枚で示され、各係数に実測/相場/仮置きのラベルがある
- [ ] GTM章のfully-loaded CAC×年間新規獲得数が財務章の新規獲得対応S&M費と差異10%以内、人員計画が財務章の採用計画と一致し、市場章のSOMと母集団が同一
- [ ] CAC・セールスサイクル・CVR・勝率が実測値で提示されている(N数付き)
- [ ] 創業者非関与の受注比率、レップramp短縮、Magic Number(0.6以上、理想1.0超)のいずれかで再現性が示されている
- [ ] 拡張戦略は1つに絞られ、型に応じた証拠(NRR/隣接1本目/attach rate等)が付いている
- [ ] 追加資金で解放されるlatent demandが具体物(未消化リード等)で示されている
- [ ] 日本エンタープライズを狙う場合、直販/パートナーの線引きとパートナーの経済的動機が書かれている

## 致命的失敗パターン

- **「全員が顧客」**: TAM全体をICPと呼ぶ。除外定義がない。→ 焦点欠如で即不合格。
- **思いつきbeachhead**: 360度の候補出しをせず、最初に思いついた(または創業者の人脈がある)セグメントで決める。ペイン深度最大の一群を検証していないGTMは、後段のCAC・CVR前提が丸ごと砂上になる。
- **ACV×motion不整合**: $10k ACVにフィールド営業、$200k ACVにセルフサーブ。ユニットエコノミクスが構造的に成立しない(Janzのmismatch)。
- **検算切断**: GTMの獲得計画と財務のトップラインが別々に作られ、突合すると矛盾する。SOMより獲得累積が大きい。
- **勝率無視のパイプライン**: 「3xカバレッジ」を勝率実測なしに引用。勝率20%なら5x必要。
- **創業者売りの外挿**: 創業者ネットワーク由来の受注をCVR実測として営業組織にそのまま外挿($1-3M ARRで停滞する典型因)。
- **拡張の同時多方面**: 隣接・地理・マルチプロダクトを全部並べ、どれにも証拠がない。
- **相場値の実測偽装**: ベンチマーク数値を自社実測のように置く。係数ラベルの欠如。
- **日本での直販一本槍**: エンタープライズ/地方中小がパートナー商流で買う構造を無視した「THE MODELをやります」宣言。

## ステージ差

- **Seed**: 逆算チェーンは仮置き係数中心で可。代わりにbeachheadスコアリングの質と、初期顧客のペイン深度の一次証拠(顧客の言葉、支払い実績、トリガーイベントの再現性)で説得する。創業者売りはむしろ正しい(Sales Learning Curve初期)。latent demandの芽(断っている需要)があれば強い。
- **Series A〜Growth**: 本要素の中心戦場。CAC・サイクル・CVR・勝率の実測、Magic Number 0.6-1.0+、創業者非関与受注、レップ増員と獲得の比例関係——「資金を入れれば同じ機械が回る」の定量証明が必須。beachheadの浸透実績(シェア・リファレンス数)と隣接1本目の初期証拠まで見せる。
- **IPO・IR**: 個別セグメントの浸透率よりも、NRR・拡張ARR比率・S&M効率(Magic Number、CAC payback)の持続性と、次の成長カーブ(第2セグメント/第2プロダクト/海外)の確度が焦点。NRR120%超は評価倍率に直結するため、拡張メカニズムの構造説明(usage-based等)を開示物に組み込む。パートナー商流比率と依存リスクの説明責任も生じる。
