# tam-sam-som — トップダウンTAM・市場からみたSAM・ボトムアップSOM

3層は役割が異なる。TAM=賞金の大きさ(トップダウン、天井の証明)、SAM=戦場の定義(市場構造からの減算)、SOM=実行計画の数値化(ボトムアップ、主役)。Sequoiaテンプレートは「TAM (top down), SAM (bottoms up), SOM」と算出方向まで指定している。投資家が30秒で見るのは3点: ズームアウトできるか(野心)、ズームインできるか(現実性)、算数が通っているか(厳密性)。

## 一本のストーリーへの接続

- 直前(課題・解決策)から: 「この課題を持つ顧客は◯◯である。**したがって**、その顧客が今払っている金額×顧客数が市場の天井であり、我々が定義する戦場はその中の◯◯である」。TAMのICP定義は課題スライドの顧客定義と一語一句レベルで一致させる。課題が「個人皮膚科医」ならTAMは「ヘルスケア全体$400B」ではない。
- 直後(GTM・トラクション・財務計画)へ: 「SOMは◯社×到達率×ACVで$◯◯。**したがって**、5年後売上$◯◯(SOM内)を、営業◯名×生産性で獲得する」。SOMはGTM計画の入力であり、財務計画の売上上限。ここが切れると市場スライドは「飾り」と判定される。
- 拡張ストーリーへ: 「初期wedgeのSAMで勝つ。**したがって**、その顧客基盤・データ・信頼を足場に隣接市場◯◯へ拡張し、TAMは$◯→$◯に広がる」。拡張は製品ロードマップと実績トリガーで裏付ける。

## ベスト構成(節立て)

1. **市場の定義文(1-2行)**: 「誰の・何の支出を置き換える/創造するか」を先に言語化。数字より先に定義。
2. **TAM(トップダウン+検算)**
   - 一次ソースの格: 政府統計・業界団体統計・上場企業開示 > Gartner/IDC/Forrester(米)・矢野経済/富士経済(日) > メディア・ブログ推計。ソース名・発行年・定義範囲をスライド上に小さく明記。
   - 三角測量: トップダウン(調査会社)×ボトムアップ(顧客数×単価)×value theoryの2つ以上で算出し、±15-30%に収まれば堅牢。乖離したら前提のどれかが誤りで、乖離原因の説明自体が戦略的洞察になる(例: 調査会社の市場定義に自社カテゴリが含まれない)。
   - 独自推計を使う場合は計算式・前提・出典を全部開示(東証ヒアリング: 独自推計は前提条件の詳細開示が条件)。
3. **SAM(構造からの減算)**
   - TAMから引く理由を1行ずつ明示: 地理(展開国のみ)、言語、セグメント(ICP該当のみ)、チャネル(直販で届く範囲)、規制(免許・法制約)、技術要件(クラウド移行済み企業のみ、等)。
   - 各減算係数に根拠を付す(「日本語対応のみ→対象は国内◯社: 経済センサス」)。減算根拠のないSAMは「TAMの言い換え」と見なされる。
   - SAM/TAM比が90%超なら絞り込み不在のサイン。逆に極端に小さいなら市場選定自体を再考。
4. **SOM(ボトムアップ・ロールアップ)**
   - 基本式: ICP該当社数(実在リストで数える)×到達可能率×受注率×ACV。B2B SaaSの初年度qualified pipeline受注率ベンチマークは2-4%。
   - キャパシティ検算: 営業rep数×rep当たり受注/四半期×4×ACV。この2式が近似しないSOMは絵空事。
   - ACVは現行プライシング(実顧客の支払額)を使う。希望価格ではない。
   - SOMは「3-5年で現実に取れる売上」として定義し、財務計画の最終年売上≦SOMを守る。
5. **(該当時)value-theory TAM / TAM拡張**
   - value theoryの使いどころ: 既存市場カテゴリが存在しない、または価格を再定義する場合のみ。立証責任は重い: (a)顧客の現行代替コスト(人件費・既存支出)の実測、(b)WTPの一次証拠(実契約価格、有償パイロット、価格テスト)、(c)価値の◯%を取るという取り分ロジック。
   - Gurley(Uber論)の市場拡大4ドライバーで論証: ①体験の実質改善(待ち時間・信頼・決済)②価格弾力性(価格低下→新需要)③新ユースケースの列挙(6個等、具体的に)④上位支出の代替(車所有$6T)。単一値でなくシナリオテーブル(保守/基本/強気で$◯B〜$◯B)で示す。
   - TAM拡張の作法: 「現在TAM(wedge)」と「拡張後TAM」を必ず分けて表示し、合算した1つの数字にしない。各拡張レイヤーに(a)製品ロードマップ上の該当項目(b)着手トリガー(初期市場でのシェア/実績閾値)(c)そのレイヤー単独の算出根拠を付す。Airbnb S-1が模範: Short-term $1.8T/Long-term $210B/Experiences $1.4Tをレイヤー別に、SAM($1.5T)は実測夜数×ADRベースで別建て。Snowflake S-1はIDC(2020年$56B→2023年$84B)+未定量のデータ共有分を自社推計$81Bと明示的に区別。
6. **視覚化**: 同心円か積み上げバー、各層1つの数字。ベン図・円グラフは不可。出典はスライド上に脚注。

## 合格基準チェックリスト(検証可能)

- [ ] TAM・SAM・SOMの3つの数字を、読み手が資料を閉じた後に暗唱できる(数字が1層1個)
- [ ] TAMは2手法以上で算出し、乖離±30%以内、または乖離理由を明記
- [ ] 全ての外部数字にソース名・年・URL/資料名が脚注化されている
- [ ] 独自推計は計算式と前提が再現可能な形で開示されている
- [ ] SAMの各減算に理由と根拠データが1行ずつある
- [ ] SOM = 実在ICPリスト数×到達率×受注率×ACV で計算されている(比率の後付けでない)
- [ ] SOMのキャパシティ検算(rep数×生産性)と需要側計算が近似する
- [ ] ACV=現行プライシング=財務モデルの単価、が三点一致
- [ ] 財務計画の最終年売上 ≦ SOM、かつSOM/SAMの暗黙シェアが説明可能(初年度10%超なら要再検証)
- [ ] TAMのICP定義が課題・GTMスライドの顧客定義と一致
- [ ] 現在TAMと拡張後TAMが分離表示され、拡張に製品・実績の裏付けがある
- [ ] 市場成長率(CAGR)の出典があり、TAM静止画でなく将来時点も示す

## 致命的失敗パターン

- **「巨大市場の1%」論法**: 獲得メカニズムの説明ゼロと同義。即不合格。
- **グローバルTAM×ローカル事業**: 日本語のみ・国内のみで世界市場を掲げる。信頼性が一撃で崩れる。
- **Stacked TAM**: ヘルスケア+フィンテック+SaaSを足して1つの数字にする。3つの別事業を1つに見せる誤魔化し。
- **ICPの不一致**: 課題スライドの顧客とTAMの母集団がすり替わる。
- **SOM>財務計画売上、またはSOMと財務計画が無関係**: 5年後売上がSOMを超える資料は自己矛盾。
- **出典なし・定義なしの調査会社引用**: 「◯◯調べ $50B」だけで市場定義が自社と合っているか不明。東証ヒアリングでも「実態と乖離した市場設定は判断不能」。
- **歴史的市場サイズへの固着**: 既存カテゴリ(タクシー$100B)をそのままTAMにし、価格低下・新ユースケースの拡張を無視(Damodaranの誤り。実績は約30倍)。逆に、拡張論を証拠なしで語るのも同罪。
- **SAM=TAM×一律係数**: 構造的減算でなく「とりあえず30%」。

## ステージ差(Seed / SeriesA〜Growth / IPO・IR)

- **Seed**: 目安 TAM $1B+/SAM $200-500M+。精緻さより論理の一貫性。SOMは実在顧客リスト+10-15件の顧客対話で裏付ける。「TAMよりbeachhead顧客」(Jim Goetz)——初期顧客の実在性が最強の市場証明。value theory型は有償パイロット1件が推計10ページに勝る。
- **SeriesA〜Growth**: TAM $5B+が期待水準。SOMは実績ベース(実受注率・実ACV・実CAC)に置き換え、コホートで penetration を実証。TAM拡張ロードマップ(wedge→隣接)を提示し、第2レイヤーの初期実績を1つ以上見せる。トップダウンとボトムアップの乖離説明が deep dive で必ず問われる。
- **IPO・IR(東証グロース/S-1)**: 「事業計画及び成長可能性に関する事項」で市場規模・算出根拠の開示が制度上求められ、年1回以上更新。機関投資家はTAMの大きさより信頼性・検証可能性を評価。独自推計は前提と出典の詳細開示が必須。S-1流の型: SAMは実測データ(Airbnbは実際の宿泊数×ADR)、TAMは拡張レイヤー別+第三者データ(SnowflakeはIDC)と自社推計を明示的に区別。上場後は市場規模の「見直し履歴」自体が誠実性のシグナルになる(定義変更時は理由を開示)。
