# market-positioning — 市場構造の分析・競合環境・自社ポジショニング

## 一本のストーリーへの接続(直前・直後の要素と「したがって」で繋がる書き方)

- 直前(課題・市場機会)から受ける: 「この課題は大きく、解決に金が流れる市場がある。**したがって**、その市場の構造——利益がどこに溜まり、誰が取り合っていて、なぜ既存プレイヤーが取り切れていないか——を示す」。市場規模(TAM)の数字を繰り返すのではなく、**市場の質**(成長率・断片化度・winner-take-most性)へ議論を一段深める。
- 直後(競争優位・moat / 成長戦略)へ渡す: ポジショニングの結論は「この土俵で勝つ要件は X・Y・Z であり、既存代替は構造的に満たせず、当社だけが満たせる」という**勝てる土俵の定義**。**したがって**次章は「その土俵を守り続ける仕組み(moat)」と「その土俵から拡張する順序(成長戦略)」を論じる。ここで土俵を定義しないまま moat を語ると、「何に対する防御か」が宙に浮く。
- 接続の検算: この章を削除したとき、直後の moat 章が意味を失うか。失わないなら、この章は市場説明の羅列になっており書き直し。

## ベスト構成(節立てレベルで具体)

**節1: 市場構造 — 利益プールと構造変化(スライド1〜2枚)**
1. バリューチェーンを描き、各段の売上プールと**利益プール**を分けて示す(Gadiesh & Gilbert)。利益の集中点は売上の集中点とずれるのが常(自動車の金融・アフター、決済のイシュイング等)。自社が獲りにいく段を明示する。
2. 集約/分散の構造要因を1行で判定: 供給側の断片化度(Gurley: 供給が寡占だと仲介を拒む/断片化市場ほどプラットフォームが成立)、規模の経済の有無、規制・免許の壁。
3. winner-take-most 判定は3条件で機械的に行う(Eisenmann/Parker/Van Alstyne): (a)ネットワーク効果が強い(n+1000人目の体験が n人目より良くなるか — Gurley)、(b)マルチホーミングコストが高い(併用が高コストか)、(c)特殊ニーズによる差別化余地が小さい。3つ揃えば「一社総取り→シェア先行投資を正当化」、揃わなければ「分散市場→ユニットエコノミクスと参入順序で勝つ」と、**判定結果が後段の資金使途の論理を決める**ことを明記する。
4. 構造変化のトリガー(規制・技術・行動変容)を1つに絞り、「なぜ今この構造が動くのか」を課題章の Why Now と接続する。

**節2: 競合環境 — 代替手段の全体像(スライド1枚)**
1. 競合定義は「顧客が当社を買わないとき実際に何をするか」(Dunford の competitive alternatives)。直接競合・間接競合・**ステータスクオ(手作業・Excel・何もしない)**の3層で列挙する。最大の競合が inertia である市場ではそれを正面から書く。
2. 各代替の「顧客が選ぶ理由」と「構造的限界」を1行ずつ。限界は主観形容詞でなく事実(価格体系、提供形態、対象顧客、規制対応の有無)で書く。
3. 「競合はいない」とは書かない(即座に信頼を失う定番 red flag)。空白地帯の主張方法は節4。

**節3: 勝てる土俵の定義と自社ポジショニング(スライド1〜2枚)**
1. Dunford の3段論法をこの順で書く: (i)この市場・この顧客セグメントで勝つための要件を3つ定義する(顧客の購買基準から導く)→(ii)既存代替はなぜその要件を満たせないか(構造的理由: ビジネスモデル・資産・インセンティブ)→(iii)自社はなぜ構造的に満たせるか(unique attributes → 顧客価値 → それを最も強く求める target segment)。
2. 大手プラットフォーマー隣接時の「Google が作らない理由」は能力でなく**インセンティブ**で答える(Helmer の counter-positioning): 参入すると既存収益を毀損する(joint NPV が負)、既存モデルへの認知的固着、経営陣の報酬構造。3つのどれに該当するかを名指しする。「気づいていないから」は答えにならない。
3. 比較表を使う場合: 行は顧客の購買基準(自社機能一覧ではない)、自社列が全部✓・競合が全部✗の表は信頼喪失。競合が勝つ行を最低1つ残し、「その行が重要でない顧客セグメントを狙う」と言えて初めて成立する。
4. ポジショニングマップ(2x2)を使う場合の作法: 軸は顧客インタビュー・失注理由・購買稟議の評価項目から採る(勝てるように逆算した軸は見抜かれる)。恣意性の検証: (a)軸を顧客の言葉で言えるか、(b)競合をその軸上に第三者データで置けるか、(c)右上が空白なのは「誰も気づいていない」からか「儲からないから」か答えられるか。混雑カテゴリで全員が右上に自社を置く図は逆効果——空いている象限に価値があることの証明の方が重要。

**節4: カテゴリ戦略 — 3つの選択肢と立証責任(スライド0.5〜1枚)**
| 選択肢 | 選ぶ条件 | 立証責任 |
|---|---|---|
| 既存カテゴリ内で差別化 | カテゴリに予算とリーダーが存在し、購買基準の一部で構造的に勝てる | 「その購買基準を最重視するセグメントが十分大きい」ことの証明 |
| 既存カテゴリの再定義(サブカテゴリ) | カテゴリの前提の一部が特定セグメントで崩れている | 前提が崩れている証拠(顧客行動データ)と、リーダーが追随できない理由 |
| 新カテゴリ創造 | 顧客が Excel・人力で代用しており比較対象が存在せず、かつ既存カテゴリの根本思想が誤り(Salesforce の "No Software" 型) | 最重い。問題自体の教育コストを負う。定着まで6〜10年(Play Bigger)。教育投資の資金計画とセットでないと語れない |

- デフォルトは上2つ。カテゴリ創造は「既存カテゴリで語ると自社価値が伝わらない」ことを検証してから。Dunford 自身も大半の企業に必要なのは既存カテゴリ内の設計だと述べる。
- ニッチ支配(Thiel 型「小さく始めて独占」)は Seed〜Series A の標準形: 初期市場は「支配できる小ささ」で切り、支配後の拡張経路(節: 成長戦略)への接続を明示する。

## 合格基準チェックリスト(検証可能な形)

- [ ] バリューチェーン上の利益プールの偏在を示し、自社が獲る段を1つ名指ししている
- [ ] winner-take-most 判定を3条件(NW効果・マルチホーミングコスト・差別化余地)で明示的に行い、判定結果が資金使途(シェア先行 or 収益性重視)と整合している
- [ ] 競合リストにステータスクオ(手作業・内製・何もしない)が含まれている
- [ ] 「勝つ要件3つ→既存代替が満たせない構造的理由→自社が満たせる構造的理由」の3段が揃っている
- [ ] 比較表/2x2の軸・行がすべて顧客の購買基準由来で、出所(インタビュー・失注分析・稟議項目)を口頭で説明できる
- [ ] 比較表に競合が勝つ行が最低1つあり、それでも勝てるセグメント論理がある
- [ ] 大手隣接時: 「参入しない理由」を Helmer 3類型(収益毀損・認知固着・報酬構造)のどれかで名指ししている
- [ ] カテゴリ戦略(差別化/再定義/創造/ニッチ支配)を1つ選び、対応する立証責任を果たしている
- [ ] 「競合不在」「唯一の」等の全称表現がゼロ
- [ ] IPO用途: 競合名・シェア・代替性が第三者統計または一次調査で裏付けられ、出典が付いている

## 致命的失敗パターン

1. **「競合はいません」**: 市場理解不足のシグナルとして即減点。ステータスクオが必ず競合。
2. **全部✓比較表**: 恣意性が一目で伝わり、以降の全スライドの信頼を毀損する。
3. **右上2x2の混雑**: 全社が同じ図を出すカテゴリで同じ図を出す。軸の出所を語れないと逆効果。
4. **TAMの再放送**: 市場章で市場規模を繰り返すだけで、利益プール・構造・質に踏み込まない。
5. **winner-take-most の無根拠な主張**: NW効果と言いつつマルチホーミングが容易(配車・フードデリバリー型)な構造を見落とし、シェア先行投資の論理が崩壊する。
6. **カテゴリ創造の安売り**: 立証責任(問題教育・6〜10年・投資体力)を負えないのに「新カテゴリ」を名乗る。
7. **「Google は気づいていない」**: 能力・認知の否定で答える。インセンティブ構造で答えない限り再質問される。
8. **機能比較で勝って土俵で負ける**: 勝つ要件の定義を飛ばし、機能の多寡だけ比較する。要件定義が先、比較は後。

## ステージ差(Seed / SeriesA〜Growth / IPO・IR)

- **Seed**: 市場の質の仮説が主役。利益プール分析は簡潔でよいが、初期ニッチ(支配できる小ささ)の定義と「顧客が今何で代用しているか」の一次観察は必須。比較表より「代替手段の構造的限界」の言語化。winner-take-most 判定は仮説として明示し、検証計画に落とす。
- **Series A〜Growth**: 立証が主役。勝つ要件3つを失注分析・受注理由の実データで裏付ける。競合との勝敗データ(win rate、置換事例)を提示。winner-take-most 判定が確度を持ち、シェア先行か収益性かの資金使途論理を接続する。2x2/比較表はこの段階から有効(軸の出所を示せるため)。
- **IPO・IR**: 客観性が主役。東証グロースの「事業計画及び成長可能性に関する事項」は市場環境(市場規模・競合環境)・競争力の源泉を継続開示項目として要求し、少なくとも年1回の更新義務がある。競合名・シェア・代替性は第三者統計か一次調査で裏付け、出典を付す。「勝てる土俵」の語りは維持しつつ、形容詞を数値(シェア推移・解約率・価格プレミアム)に置換する。カテゴリ創造の物語は、審査・機関投資家には既存分類(業種・比較対象企業=バリュエーションのcomps)との対応表を併記しないと通らない。
