# financial-ask — 財務計画・調達額・資金使途・マイルストーン・資本政策・ラウンド間連続性

エクイティストーリーの終着点。ここまでの全要素(課題→解決→市場→moat→トラクション→チーム)が「だから今、この金額が要る」に収束しなければ、この節は書けない。

## 一本のストーリーへの接続(「したがって」の連鎖)

- 直前(トラクション/実行計画)から受ける: 「前ラウンドで約束したXを達成し、リスクA(例: PMFリスク)を潰した。**したがって**、残る最大リスクはB(例: 再現性ある獲得チャネル)である」— Andreessenの「リスクの玉ねぎ」: 各ラウンドは特定のリスク層を剥がすための資金であり、調達額はそのリスク除去に必要な額で決まる。
- 本要素の内部: 「リスクBを潰した状態とは、次ラウンド投資家が求める状態C(数値)である。**したがって**、Cに到達するマイルストーン群と、それに要する人員・期間・資金がこれだ」。
- 直後(クロージング/ビジョン再提示)へ渡す: 「この資金でCに達すれば、次ラウンドは○○を語る調達になる。**したがって**、今回のラウンドは△△(長期ビジョン)への必然の一歩である」。
- 禁止: ここで初出の成長率・新市場・新プロダクトを出さない。financial-askに書けるのは、前段の物語で既に根拠を示した打ち手だけ。

## ベスト構成(節立て)

### 1. 前ラウンドの総括(2番目以降のラウンドでは必須)
- 「調達額 / 当時の約束 / 達成・未達 / 未達の原因と学び」を1枚で。NFXの「証明のはしご」: 前ラウンド時点からの進捗を明示的に積み上げて見せる。未達を隠すとDDで必ず露見する。

### 2. 次ラウンド要件の提示(逆算の起点)
- a16z: 「今Series Aを調達するなら、Series B投資家へのピッチを先に考えよ」。次ラウンド投資家が見たい状態を数値で置く。相場(B2B SaaS、複数出典の幅で記載):
  - **Seed→A**: ARR $1M+が従来の目安、2024–25年はARR $2–2.5M中央値・YoY 2〜3x・NRR 100%+・burn multiple 2x未満が競争水準(CRV/SaaStr/Carta)。月次成長7–15%を6ヶ月以上(NFX: 10%+/月を最低線)。
  - **A→B**: ARR $5–10M・YoY 50–100%・NRR 110%+(110%未満はDD途中で静かに落ちる)・burn multiple 1.5x以下が目安。Bは成長率よりも耐久性(NRR・効率)を見る。
  - 日本円感覚: シリーズA調達1〜3億円/プレ5〜10億円、シリーズB 3〜6億円台が歴史的中央値レンジ(Coral Deal Terms。年次版で更新すること)。
- 非SaaSはこの表をそのまま使わず、「次ラウンド投資家が消したいリスク」を同業の直近調達事例から特定して置き換える。

### 3. 調達額の逆算計算(手順を見せられる状態にする)
1. 次ラウンド要件(§2)を達成するマイルストーン3〜5個を日付つきで置く。
2. マイルストーン達成に必要な人員計画を月次で引く(バーンは人件費駆動。採用ラグ・オンボード期間込み)。
3. 月次バーン×期間+一時費用=必要額。期間は「マイルストーン達成予定+調達活動6ヶ月」で**18ヶ月前後**(12未満=切迫シグナル、24超=過剰調達・希薄化の無駄)。
4. バッファはベアケース(§6)で検算: ベアでも次ラウンド開始時に残ランウェイ6ヶ月以上(NFX)。
5. 出た額を相場(§2の調達額レンジ)と突合。大きく外れるなら、マイルストーン設計か人員計画に戻って直す。「レンジ調達(X〜Y億円)」を語る場合はYでもXでも成立するマイルストーン2案を持つ。
- バリュエーションは指値しない。希薄化許容幅(後述の10–20%)と調達額から暗黙のレンジが決まることは理解した上で、価格は市場(複数タームシート)に付けさせる。

### 4. 資金使途 = 「カテゴリ × 潰すリスク × 達成状態」の3列対応表
- 費目の羅列(人件費60%/マーケ25%/その他15%だけ)は不合格。各カテゴリを必ずリスクとマイルストーンに接続する:

| 使途カテゴリ(配分) | 潰すリスク | 達成状態(日付・数値・観測方法) |
|---|---|---|
| プロダクト/エンジニア採用 45% | 単一プロダクト依存 | 2027/6までに新モジュールGA、既存顧客の30%が有償利用(請求データ) |
| セールス 30% | 創業者売り依存 | 2027/3までにAE 3名が各々四半期ノルマ達成(CRM実績) |
| CS/基盤 15% | 解約リスク | NRR 115%を2四半期連続(コホート表) |
| 予備 10% | 期間リスク | ベアケースでもランウェイ18ヶ月維持 |

- マイルストーンの書式: **[日付] + [数値] + [観測方法(何のデータで検証するか)]**。「プロダクト強化」「採用加速」のような観測不能な文言は書き直し。
- 「このラウンドで潰す最大リスク」を冒頭1行で宣言し、配分最大のカテゴリと一致させる。
- 約束は控えめに置き、実績で上回る(NFX: overpromiseではなくexceed expectations)。ピッチ中に会うVCへの月次アップデートで進捗を先に見せると約束の信頼性が上がる。

### 5. 財務計画とのtie-out(姉妹スキル startup-financial-modeling とのインターフェース)
- 原則: **物語にない成長率をモデルに入れない/モデルにない打ち手を物語に書かない**。投資家は予測の的中でなく「物語と数字の一致」を見る(数字の不一致は即レッドフラグ)。
- 検算手順(双方向、各項目でストーリー本文とモデルのセル値を突合):
  1. モデルの売上成長の各ドライバー(顧客数×単価×継続率の分解)→ その成長を生む打ち手がストーリーのどの節にあるか指差し確認。孤児ドライバーは削除。
  2. ストーリーの打ち手(新チャネル、新プロダクト、値上げ、海外)→ モデルのどの行に反映されているか。孤児の打ち手は本文から削除。
  3. 定義の統一: 「顧客数」「ARR」等の用語定義を1つに(「500顧客」vs「550アカウント」の不一致がDDを数週間止めた実例あり)。
  4. マイルストーン日付とモデルの四半期値の一致(例: 「2027/3にARR3億」ならモデルの該当セルが3億)。
  5. 調達額=モデルのキャッシュ最低残高が正になる最小額+バッファ、をキャッシュシートで確認。
- **エクイティストーリーが財務モデルに渡すドライバー一覧**(このリストを埋めてから modeling スキルに渡す):
  1. 顧客獲得: チャネル別の新規顧客数/月とその根拠(現在の実測CVR・パイプライン)
  2. 単価: 初期ACV、値上げ・アップセルの時期と幅(打ち手に対応)
  3. 継続: ロゴチャーン率、NRR(コホート実測を起点)
  4. 人員計画: 職種別ヘッドカウントの月次追加(マイルストーンに対応)と単価
  5. 一時費用: マイルストーン固有の投資(認証取得、海外拠点等)
  6. 調達: 今回額・想定時期、次ラウンドの想定時期・前提バリュエーションレンジ
  7. ケース分岐: ベース/アップ/ベアで動かす変数と幅(§6)
  8. 資本政策: 現キャップテーブル、SAFE/J-KISS残高とキャップ、プール拡張予定

### 6. 3ケース設計(ベース/アップサイド/ベア)
- **ベース**: 調達額とマイルストーンの根拠。実測トレンドの素直な延長+資金による加速。ピッチの主役はこれ1本。
- **アップサイド**: 「星が揃えばこの市場でこの規模」を示す1枚。ドライバーは既出の打ち手の上振れ(例: NRR 125%、チャネルCVR上振れ)のみ。TAMの言い換えにしない。
- **ベア**: 投資家に見せる用ではなく耐久性の証明用。売上5割減・採用半減でも(a)コアマイルストーン1–2個は達成、(b)ランウェイ18ヶ月維持、(c)ブリッジなしで次ラウンドに立てる、を示せる構造にする。聞かれたら即答できる状態が合格線。
- 語り方: 「ベースで次ラウンド要件に届く。アップサイドはこの資金の上限価値。ベアでも死なない」。3本を等価に並べない(自信のなさに見える)。

### 7. 資本政策(1枚)
- 現在: 創業者持分・従業員プール・投資家別持分。SAFE/J-KISS/CBは転換後fully-dilutedで表示(キャップ・ディスカウント込み)。
- 今回ラウンド後: 希薄化率、プール拡張(プレかポストか)込みのプロフォーマ。
- 相場: 1ラウンドの希薄化は**10–20%**が日本の目安(Coral: Series A中央値〜15%、B〜20%)。米国はSeed 20%前後・Series A 18–20%+プール10–15%(Carta)。Series A完了時点で創業者チーム50%超残存が健全ラインの通例。
- 資本政策は不可逆。シードでの取りすぎ(A時点で創業者30%以下)は以後全ラウンドの構造問題になる旨を明記し、逆算表(IPO時点の経営陣持分から遡る)を持つ。

## 合格基準チェックリスト

- [ ] 調達額が「次ラウンド要件→マイルストーン→人員計画→月次バーン×期間」の逆算で説明でき、計算過程を口頭で再現できる
- [ ] ランウェイが12〜24ヶ月レンジ内(目安18ヶ月+調達活動6ヶ月)で、根拠がモデルのキャッシュシートと一致する
- [ ] マイルストーンが3〜5個、すべて「日付+数値+観測方法」で書かれ、観測不能な動詞(強化・加速・推進)がゼロ
- [ ] 「このラウンドで潰す最大リスク」が1行で宣言され、資金使途の最大配分カテゴリと一致する
- [ ] 資金使途の全カテゴリが3列対応表(カテゴリ×リスク×達成状態)で埋まり、費目のみの行がない
- [ ] 各マイルストーンが、達成時に次ラウンド投資家のスクリーニング基準(ステージ別相場)を満たす水準に設定されている
- [ ] 前ラウンドの約束と達成/未達が1枚で開示され、未達には原因と打ち手の変更が付いている(初回ラウンドは免除)
- [ ] モデルの全売上ドライバーに対応する打ち手が本文にあり、本文の全打ち手がモデルの行にある(双方向tie-out完了)
- [ ] ピッチ資料・モデル・データルームで顧客数/ARR等の定義と数値が一致している
- [ ] ベアケースで (a)コアマイルストーン達成 (b)次ラウンド開始時ランウェイ6ヶ月以上 が成立する
- [ ] fully-diluted資本政策表があり、今回希薄化が10–20%レンジ(逸脱時は理由を明記)、SAFE/J-KISS転換が織込み済み
- [ ] バリュエーションの指値がどこにも書かれていない(希望として口頭でも言わない準備ができている)

## 致命的失敗パターン

1. **費目羅列の使途スライド**: 円グラフ+人件費60%で終わり。何のリスクを潰すか不明 → 「金の使い方を考えていない」と読まれる。
2. **tweener化**: 中途半端な額を調達し、次ラウンド要件に届かない中間状態で資金切れ。シードには大きすぎAには足りない実績でブリッジ調達に追い込まれる。原因はほぼ「相場からの調達額決定」(逆算の欠如)。
3. **物語と数字の乖離**: デッキの成長率とモデルの成長率が違う、本文にない新規事業がモデルの後半成長を支えている → DDで信頼が壊れ、以後全数値が疑われる。
4. **バリュエーション指値**: 高い指値はアンカーとなり交渉を殺し、低い指値は上限になる。市場に価格を付けさせる原則の放棄。
5. **前ラウンド未達の隠蔽**: 登記簿・既存投資家リファレンスで必ず露見。未達自体より隠蔽が致命傷。
6. **ランウェイ12ヶ月未満での着手**: 切迫が価格交渉力を消す。NFXの目安: クローズ時に残6ヶ月を切らない=調達開始は残12ヶ月以上で。
7. **希薄化の先送り無視**: SAFE/J-KISSを額面で足し、転換後持分を計算していない。シリーズAで創業者持分が想定より10pt低いことが判明する典型事故。
8. **過剰調達の自慢**: 24ヶ月超の資金は規律喪失と次ラウンドバリュエーションのハードル引き上げのシグナルとして読まれる。

## ステージ差

- **Seed**: 潰すリスクはPMF。マイルストーンは売上よりも「PMFの観測可能な証拠」(コホート維持率、オーガニック比率、有償継続)。調達はJ-KISS/みなし優先株(日本)・SAFE(米国)が主流で、バリュエーション議論を後送りできる代わりに、キャップ設定=実質的な価格であることと累積希薄化を資本政策表で管理する。日本シードの希薄化目安10–20%。
- **Series A〜Growth**: 潰すリスクは「再現性(A)→スケール効率(B)→市場支配(C)」へ移る。数値要件(§2相場)が明確化し、前ラウンド総括が必須になる。日本は参加型優先株が慣行(2019年時点で約97%。米国はほぼ非参加型)のため、優先分配の累積がM&A時の創業者手取りを圧縮する点を資本政策のリスクとして把握。ラウンドごとに希薄化15–20%、優先株条件(1x・参加型か否か・ラチェット)の連続性を管理する。
- **IPO・IR**: 「調達額の逆算」は「手取金の使途」の開示に置き換わる。目論見書の資金使途は具体性とロジックが評価軸(待機資金なら投資方針・代替使途まで開示)。マイルストーンは業績ガイダンスと「事業計画及び成長可能性に関する事項」(グロース市場の継続開示)に変わり、「控えめに約束し実績で上回る」原則がガイダンス設計にそのまま適用される。資本政策は希薄化管理からオーバーハング(ロックアップ、VC売出し)と流動性設計に主題が移る。
