# Clarity-First Composition

Use this contract to make the governing relationship immediately understandable before adding visual material.

## One Relationship

Freeze one `primary_relationship`: comparison, sequence, hierarchy, transformation, or loop. Select one matching visual grammar and use it across the complete body. Supporting evidence stays inside that grammar through aligned rows, columns, labels, or values.

Use 2-4 major regions. Give each region one semantic role and one visual role. Establish one start, one ordered path, and one landing. At thumbnail size, a reviewer can point to all three within three seconds.

## Element Economy

Give every visible element one explanatory job. Use these production budgets:

- primary relationship: 1
- visual grammar: 1
- visual anchor: 1
- emphasis system: 1
- connector set: 1
- connectors: up to `major_region_count - 1` for a one-way sequence
- structural rules: the shared baselines or boundaries required by the selected grammar
- icons: one consistent family, used when an icon shortens recognition
- conclusion: integrated into the landing region or one aligned takeaway row

A loop composition uses one closed path as its primary grammar. A sequence composition uses one forward direction. A comparison composition uses common baselines and aligned columns. A hierarchy composition uses shared levels and parent-child alignment. A transformation composition uses a clear before / mechanism / after path.

## Region Independence And Softness

Create coherence through shared grid lines, repeated spacing, matched heights, and a direct connector. Use 2-4 independently legible regions. Select a common outer frame only when containment is the primary relationship. Otherwise, let quiet gutters separate the regions and let alignment create unity.

Use 8px or 12px radii for major region surfaces and 4px or 8px for repeated modules. Keep one radius within each peer group and preserve straight rules and connector centerlines. This creates a precise structure with a small amount of warmth.

## Luminous Tone Vocabulary

Use lightness as a semantic vocabulary through flat, quiet surface steps:

- ambient light — `slide_background` semantic role
- lifted light — independent-region `neutral_panel` semantic role
- structural light — mechanism or evidence `structural_panel` semantic role
- focal light — one conclusion or decision `single_focus_fill` semantic role

Assign one tone to one meaning and repeat it consistently. Bind `ambient` to the canvas, `lifted` and `structural` to named regions, and `focal` to one named decision or conclusion content atom. Pair each tonal role with a label, position, or structural boundary so the distinction remains legible in grayscale.

Record `tone_plan` with `ambient_role`, `region_role`, `structural_role`, `focus_role`, `semantic_tone_count`, `focal_tone_count`, `tone_role_consistency`, `flat_tonal_layers`, and four `role_assignments`. The four tone fields hold stable semantic roles rather than hex values, so a palette swap remains a single-file operation. Each assignment records `role`, `target_type`, and `target_id`. Approve 3-4 semantic tones, one focal tone, role consistency >=0.95, flat tonal layers, and exactly one valid target for each role.

## Clarity Plan

Record `clarity_plan` with `primary_relationship`, `visual_grammar`, `major_region_count`, `region_roles`, `start_region_id`, `landing_region_id`, `reading_path`, `connector_count`, `connector_set_count`, `visual_anchor_count`, `emphasis_system_count`, `outer_frame_role`, `redundant_encoding_count`, `mark_language_consistency`, `region_internal_distribution_consistency`, and `thumbnail_path_status`.

Approve one connector set, one anchor, one emphasis system, zero redundant encodings, one mark language with consistency >=0.95, peer-region internal distribution consistency >=0.90, an explicit start and landing, and `thumbnail_path_status: approved`. A sequence connector count stays at or below `major_region_count - 1`; a loop declares one closed path instead.

Record `surface_plan` with `top_level_surface_count`, `major_radius_px`, `peer_radius_px`, `radius_consistency_score`, and `grouping_methods`. Approve 2-4 top-level surfaces, the defined radius tokens, radius consistency >=0.95, and grouping through shared axes plus repeated spacing or connectors.

## Review And Repair

Review the slide in this order:

1. Name the relationship visible in the body.
2. Point to the start, path, and landing.
3. Confirm each boundary, rule, connector, icon, and highlight directly supports that path.
4. Merge repeated evidence that carries the same meaning.
5. Recompose the slide when more than one visual grammar competes for attention.

The approved result feels simple because meaning, alignment, and emphasis all point in the same direction.
