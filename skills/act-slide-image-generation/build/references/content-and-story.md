# Content And Story

Use this file for brief-to-deck reasoning.

## Deck Contract

Resolve purpose, audience, required decision, intended use, language, source strictness, and palette. Use embedded ACT defaults when the brief supports them; clarify decisions that materially change the result.

## Story Logic

1. State one governing thought.
2. Map setup, tension, proof turn, and landing.
3. Write action titles whose read-through forms one argument.
4. Give every slide one primary focal object and one dominant structure.
5. Match every title to a named exhibit element that proves it.
6. Use MECE support and declare inductive or deductive body logic.

## Ordinal Semantics

Apply `ordinal_semantics_lock` before adding `01/02/03`, `Step N`, circled numbers, or numbered badges. Use ordinals when order itself carries meaning: a timeline, phase roadmap, staged flow, ranked list, deductive premise chain, swimlane, or causal staircase. Use parallel position, alignment, and semantic labels for MECE comparisons, capability groups, KPI strips, matrices, and independent pillars. When numbered elements can be reordered without changing the claim, repair by removing the ordinals and strengthening grouping or relationship labels.

## Evidence And State

Classify load-bearing content as fact, estimate, assumption, anecdote, or open. Render actuals with solid emphasis and future/plan/target states with pale or dashed treatment plus labels. Use published, independently traceable sources for visible footers; keep internal inputs in the source ledger and speaker notes.

## Density

Choose a density tier before layout. Increase decision value through comparisons, benchmarks, units, denominators, assumptions, annotations, evidence strips, and supporting context. Preserve one reading path and >=20pt body text. Use the footprint targets in `core-layout-and-typography.md` so evidence occupies the canvas confidently.

## Visual Grammar

Choose the structure that makes the claim easiest to see: comparison, table, flow, roadmap, loop, matrix, KPI bridge, evidence strip, architecture stack, or restrained illustration. Use icons as sparse wayfinding. Use Honey as a selective decision signal and Petrol as the stable structural accent.
