# システム設計 & アーキテクチャ (Architecture Docs)

このディレクトリは、本プロジェクトにおけるシステム設計、インフラ構成、および各種アーキテクチャの意思決定レコード（ADR）を管理するための場所です。

## 📁 今後のドキュメント追加予定
将来的に、プロジェクトの規模拡大に伴って以下のようなドキュメントを追加・保管します。

- **`system_overview.md`**: システムの全体像、データフロー、およびコンポーネント連携図。
- **`infrastructure_blueprint.md`**: クラウドプロバイダ（GCP/AWS等）上のインフラ構成やコンテナ設計。
- **`decisions/` (Architecture Decision Records)**: 技術選定やプロトコル選定に際して行われた決定（ADR）の一覧。
- **`security_model.md`**: 暗号化、認証（OIDC/AppRole）、認証認可モデルなどのセキュリティ定義。

## 🛡️ 設計の基本方針
1. **クラウドネイティブ & セキュアファースト**: すべてのインフラ構成は暗号化通信を原則とし、シークレットは HashiCorp Vault により動的に管理します。
2. **高可用性（HA）**: 本番環境におけるシステムは単一障害点（SPOF）を排除し、冗長性と耐障害性（Raft Integrated Storage等）を考慮して設計されます。
3. **疎結合・イベント駆動**: 各マイクロサービスやエージェントは明確な境界コンテキスト（Bounded Context）を持ち、疎結合に連携します。
