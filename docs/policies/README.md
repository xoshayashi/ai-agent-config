# チーム規約 & セキュリティポリシー (Policies & Guidelines)

このディレクトリは、開発チームにおけるコーディング規約、ガバナンス、デプロイポリシー、およびセキュリティポリシーを管理するための場所です。

## 📁 今後のドキュメント追加予定
プロジェクトの拡大に合わせて、以下のようなガイドラインや規約ドキュメントを配置します。

- **`coding_standards.md`**: 言語ごと（Go, Python, TypeScript等）のフォーマッタ設定、静的解析ルール。
- **`git_flow_policy.md`**: ブランチ運用ルール（GitHub Flow/Git Flow）、コミットメッセージのプレフィックス規格。
- **`security_compliance.md`**: 機密情報の保護（Vault利用規約）、データ消去ポリシー、個人情報（PII）の取り扱い。
- **`review_guidelines.md`**: プルリクエスト（PR）のレビュースタンダード、自動マージポリシー。

## 🛡️ チームの基本ルール
1. **コミット時シークレット混入の厳禁**: APIキー、秘密鍵、パスワード等は、いかなる理由であってもリポジトリ（Git履歴）にコミットしてはなりません。必ず環境変数、または HashiCorp Vault を経由して動的に取得します。
2. **自動CI/CDパスの必須化**: メインブランチ（`main` / `master`）へのマージは、必ずリンター、静的解析、およびユニットテストが100%合格している必要があります。
